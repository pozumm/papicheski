Звіт з першого Work Case 

Git — це система керування версіями керування коду серед розробників, за допомогою нього розробники можуть легко вносити зміни
в спільний проєкт.Також допомагає легко організовувати розробку різних проєктів.
Основні дії в Git:
git init — створення нового репозиторію;
git clone — клонування існуючого репозиторію;
git add — додавання файлів до індексу для наступного коміту;
git commit — збереження змін у репозиторії з повідомленням;
git push — відправка змін на віддалений репозиторій;
git pull — отримання останніх змін з віддаленого репозиторію;
git status — перевірка стану репозиторію (які файли змінені або додані).
Комміт — це збереження змін у репозиторії.Він використовується для   відслідковування точних змін час також хто змінював та відкотитись у разі поломки.
